android indicator解析

2017-07-19
androidgithub
anslysis, github, indicator

解析对象

  • BGAIndicator-Android 滑动指示器

  • PagerSlidingTabStrip Android-导航栏特效，新闻类APP(仿iOS 版的网易，今日头条效果)，主要是导航栏字体大小和颜色的渐变特效

  • ViewPagerIndicator Indicator 取代 tabhost，实现网易顶部tab，新浪微博主页底部tab，引导页，无限轮播banner等效果，高度自定义tab和特效

  • ColorTrackTabLayout 一个自适应Tab宽度，可以滑动文字逐渐变色的TabLayout

  • ColorTrackView 字体或者图片可以逐渐染色和逐渐褪色的动画效果

    ​

BGAIndicator-Android

支持Indicator和ViewPager宽度不相等时也能正确运行的滑动指示器

  • 效果图

  • 自定义属性说明

  • 功能

    • 主要实现单屏viewpager滑动指示器展示
    • 不支持指示器超过一屏的滑动，（包裹HorizontalScrollView后，跟随页面切换，滑动指示器的联动亦是问题）

  • 实现

    • 继承自LinearLayout

    • 监听ViewPager滑动事件，实现滑动指示器绘制

    • 初始化添加tabs

    • 切换tabs

    • 绘制滑动下划线

    • 监听切换页面

PagerSlidingTabStrip

Android-导航栏特效，主要是导航栏字体大小和颜色的渐变特效
可以是固定的几个,可也可以是水平滚动

  • 效果图

  • 自定义属性说明

  • 功能

    • 基本实现类似头条滑动指示器，可多屏滑动展示

  • 实现

    • 继承HorizonalScrollView

    • 监听viewpager滑动，绘制指示器

ColorTrackTabLayout

一个自适应Tab宽度，可以滑动文字逐渐变色的TabLayout,该控件继承自TabLayout做了一些扩展。

  • 效果

  • 自定义属性

  • 使用

      titles = new String[]{"推荐", "视频", "社会"};
  final List<Fragment> fragments = new ArrayList<>();
  for (int i = 0; i < titles.length; i++) {
      fragments.add(MyFragment.newInstance());
  }

  mViewPager.setAdapter(new FragmentPagerAdapter(getSupportFragmentManager()) {
      @Override
      public Fragment getItem(int position) {
          return fragments.get(position);
      }

      @Override
      public int getCount() {
          return titles.length;
      }

      @Override
      public CharSequence getPageTitle(int position) {
          return titles[position];
      }
  });
//隐藏指示器
mTab.setSelectedTabIndicatorHeight(0);
//设置每个Tab的内边距
mTab.setTabPaddingLeftAndRight(20, 20);
mTab.setupWithViewPager(mViewPager);

  • 实现

    继承自TabLayout

    既然是随着页面的滑动文字颜色渐变那么肯定少不了ViewPager的页面监听，这个在我们调用setupWithViewPager的时候TabLayout就已经添加监听。

​ 创建了mPageChangeListener并添加了监听

mPageChangeListener = new TabLayoutOnPageChangeListener(this);，所以我们必须要重写setupWithViewPager删除掉原来的监听，换成我们自己的监听

​ 还需要做的一点就是把TabLayout每一个的Tab布局替换成我们的。怎么替换呢？重写addTab，在添加的时候改成我们的布局

​ 其关键就是将我们的布局利用setCustomView方法来设置上去。
